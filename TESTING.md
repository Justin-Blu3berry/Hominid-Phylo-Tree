# Evidence of Correctness

Include at least one of the following (more is better):
Automated tests (e.g., in a tests/ directory) with a brief description in the README or a TESTING.md file.
Comparison to a baseline or known result (e.g., comparison to a standard tool on a small dataset).
Clear sanity checks or invariants that your algorithm satisfies (and documentation of how you checked them).
You do not need a perfect benchmark, but you must show that you thought carefully about how to evaluate correctness.
